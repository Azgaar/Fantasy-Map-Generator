# State Taxes & Treasury

States accumulate currency in `state.treasury` from two taxes generated by `States.collectTaxes()` once per production cycle. The economy is otherwise frozen â€” there are no recurring ticks, no spending, no compounding.

## Tax sources

### Sales Tax (`state.salesTax`)

A rate in `[0, 1]` applied to deals where the state is the seller.

- **Local sale (burg â†’ market)**: `Markets.sell` resolves the seller burg's state rate and records `deal.tax = units Ă— price Ă— rate`. The production module deducts that amount from gross revenue before crediting `burg.treasury`. `States.collectTaxes()` sums `deal.tax` into the seller's state treasury.
- **Global sale (market â†’ market)**: `Markets.runGlobalTrade` adds `exporterTaxPerUnit = exporterState.salesTax Ă— exporterPrice` to the landed cost. Trades that become unprofitable under the friction are skipped. Executed trades record `deal.price = landedCost` and `deal.tax = exporterTaxPerUnit Ă— units`. High-tax states therefore become poorer exporters.

### Poll Tax (`state.pollTax`)

A flat per-population-point levy applied once per cycle: `revenue = state.pollTax Ă— (state.rural + state.urban)`. No deduction from any burg or rural source â€” the money simply appears, matching the frozen-cycle economy.

## Rate generation

Rates are seeded from `state.form` at state form assignment (in `States.defineStateForms`) and jittered per state via `gauss(base, base Ă— 0.15, base Ă— 0.5, base Ă— 1.5)` rounded to 2 decimals.

| Form | Sales Tax base | Poll Tax base |
|---|---|---|
| Monarchy | 0.15 | 0.20 |
| Theocracy | 0.25 | 0.10 |
| Union | 0.07 | 0.13 |
| Republic | 0.05 | 0.15 |
| Anarchy | 0.00 | 0.00 |

Anarchy collapses to exactly 0. Neutrals (state 0) always have rates 0 and treasury 0.

## Collection (`States.collectTaxes`)

Called from `public/main.js` after `Production.produce()` and `Markets.runGlobalTrade()`:

1. Reset every non-neutral state's `treasury` to 0.
2. For each `deal` with a `tax` field, look up the seller state (burg's state for burg sellers; the market's center burg's state for market sellers) and credit `state.treasury += deal.tax`.
3. For each non-neutral state, add `pollTax Ă— (rural + urban)` to its treasury.

This is the only writer of `state.treasury` outside the editor (which can also write via the Treasury dialog manual override).

## Editor surface

The States Editor exposes a Treasury column. Clicking a treasury value opens a dialog with editable `salesTax`, `pollTax`, and `treasury` fields plus a read-only revenue breakdown. Rate edits do not retroactively change deals â€” they take effect on the next production regeneration.

## Backward compatibility

When loading an older save (`auto-update.js`), the migration runs `States.defineTaxRates(state)` for every non-neutral state and resets `treasury = 0`. The migrated rates are jittered exactly like newly generated states (same gauss formula on the form-based base) â€” so the same world reloaded twice keeps stable rates after the first migration, but loading a pre-1.123 file produces rates that differ from the legacy hard-coded `0.2` salesTax. Any custom per-state tax edits made under the old regime are overwritten.

## Out of scope

- Recurring economy ticks / yearly cycle accumulation.
- Spending (administration cost, military upkeep, corruption).
- Per-`formName` modifiers or per-state-type modifiers.
- Import-side tariffs (only exporter-side sales tax is applied to inter-market trade).
